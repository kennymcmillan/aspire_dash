How this gets into aspire_dash

  1. Edit this file. Tailwind utilities are unlimited; DaisyUI gives semantic classes (`btn`, `card`, `badge`, `tab`).
  2. When a prototype looks great, copy the class string (e.g. `border border-slate-200 border-l-4 border-l-[#004185] rounded-lg px-5 py-3.5 shadow-sm`).
  3. Convert to a single CSS rule in aspire_dash/assets/00_aspire_base.css with a class name like .kpi-tile.
  4. Update the Python helper (e.g. aspire_dash/components/kpi.py) to emit className="kpi-tile accent-aspire".
  5. Sync to demo/assets/00_aspire_base.css (setup_app only copies once).
  6. Bump version + CHANGELOG. Every consumer app picks up the new look on next SHA bump.
